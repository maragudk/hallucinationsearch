### Security / escaping

Spot-checked that `og:title` and `og:description` are properly HTML-escaped. `curl -s 'http://localhost:8091/?q=%3Cscript%3Ealert(1)%3C/script%3E'` yields `og:title="Hallucination Search: &lt;script&gt;alert(1)&lt;/script&gt;"` (and likewise for description). Double-quote in the query (`?q=a%22b`) renders as `&#34;`. gomponents' attribute escaping is doing the right thing; no injection surface on the OG tags.

## Judgment calls

- **Exported `AbsoluteURL` vs unexported `absoluteURL`.** The spec suggested lowercase. The builder exported it so the five scheme/host/forwarded-proto branches can be unit-tested directly rather than indirectly through `HomePage`/`ResultsPage`. Accepted -- direct tests make scheme-resolution failures much easier to diagnose, and `AbsoluteURL` is a generally useful primitive that any future page or handler might want. Not a blocker either way.
- **`og:url` on an empty `?q=`.** When a user hits `http://localhost:8091/?q=`, the handler routes to `HomePage` (because `strings.TrimSpace(raw) == ""`), so `og:title`/`og:description` are the home-page copy. But `og:url` still contains the literal `?q=` because `AbsoluteURL` takes `r.URL.RequestURI()` verbatim. Technically this means sharing `/?q=` and sharing `/` produce subtly different canonical URLs in the card. Cosmetic at worst -- crawlers treat them as different URLs but the unfurl content is identical. Not worth fixing.
- **No `og:locale`, `og:image:alt`, `twitter:site`, `twitter:creator`.** Builder deliberately stuck to the spec's enumerated 8+4 tags. `og:image:alt` would be a minor a11y win for screen readers inside social previews; leaving it out is defensible (site is English-only, no single hard-coded alt text fits both the home card and the per-query results card). Worth revisiting when a11y gets a dedicated pass. Not a blocker.
- **`props.R == nil` guard protects against nil request but not nil `props.R.URL`.** In practice all HTTP handlers produce a non-nil `URL`, so this is theoretical. The `TestErrorPageSkipsOGWhenRequestIsNil` case covers the real-world nil path. Not a blocker.
- **Non-https `X-Forwarded-Proto` values are all treated as http.** The code accepts only the literal string `"https"`. A proxy that sent `HTTPS` (uppercase) would be misread as http. Unusual enough to ignore -- standard reverse proxies send lowercase -- but worth flagging if this ever runs behind something weird. Not a blocker.
- **`og:image` is not hashed.** Deliberate, as the builder's diary explains: crawlers cache the URL verbatim, and hashing would invalidate every already-cached share card on every deploy that happens to touch the JPEG bytes. Spec-compliant. The downside is that if the image is ever genuinely replaced, cache-busting requires a manual `?v=N` append. Acceptable tradeoff.
