## Blocking issues

### 1. `/site/...` returns 500 instead of 502 when the poll times out

**File:** `http/search.go` (lines 288-305 before my fix)

The poll loop uses `select { case <-pollCtx.Done(): ...; case <-ticker.C: GetWebsite(pollCtx, ...) }`. When `pollCtx` expires, if `select` picks the ticker branch first, the subsequent `GetWebsite` call runs with an already-cancelled context and returns `context.DeadlineExceeded`, which is **not** `model.ErrorWebsiteNotFound`. The handler then falls into the "internal error" branch and sends a misleading HTTP 500 with `"internal error"` body, instead of the intended 502 `"website not ready"`. The app.log confirmed this happened in my manual test (`level=ERROR msg="Error getting website" error="context deadline exceeded"`).

**Fix applied in the worktree:** I added an explicit check for `context.DeadlineExceeded`/`context.Canceled` before falling through to the generic error branch, so the handler returns 502 in that case. Build + lint + test pass with the change.

### 2. Skeleton-to-filled card flash on cached pages

**File:** `html/results.go`

Both the skeleton div and the filled div are rendered server-side with only Datastar's `data-show` directive controlling visibility. Until the Datastar JS loads and evaluates, the browser shows both divs. The skeleton has `animate-pulse`, so it's particularly visible. The builder flagged this concern in the diary.

**Fix applied in the worktree:** add a conditional inline `style="display: none"` on whichever div we know server-side should be hidden (skeleton when `haveResult`, filled when `!haveResult`). Datastar then toggles `style.display` as expected once it initialises. Verified the output HTML now contains exactly 10 `style="display: none"` attributes either way.

## Non-blocking issues

### CSP permits inline scripts on `/site/...` pages

The global CSP sets `script-src 'self' 'unsafe-inline'` (controlled by `CSP_ALLOW_UNSAFE_INLINE=true` in `.env`). The Opus prompt forbids `<script>` tags and all observed outputs comply, but the policy would let an inline script execute if the model ever slipped one through. Since the app is behind an auth proxy and the LLM is trusted, I'm leaving this as a nit. Cheap follow-up: add a route-level middleware on `/site/` that sets `script-src 'none'; base-uri 'none'` and tightens `default-src 'none'`. Leaving this for the builder to decide.

### `.github/workflows/ci.yml` and `compatibility.yml` still spin up `versitygw`

The s3/versitygw dependency was removed from the code and `docker-compose.yml`, but the CI and nightly compatibility workflows still declare it as a service. Harmless (the tests no longer touch it) but dead weight. Drop the `services:` blocks to tidy up.

### Future nit: SSE connection can race a just-upserted query row

`handleEvents` returns silently when `GetQueryByText` returns no rows. In theory a fast browser could hit `/events?q=foo` before the `/?q=foo` upsert commits; in practice the main page handler has already done the upsert before rendering, so it's fine. Worth keeping in mind if SSE ever moves before the page render.

### `gai` indirect dependency not `// indirect`-free

`go.mod` lists `maragu.dev/gai` under the indirect block even though `llm/llm.go` imports it directly. A `go mod tidy` would move it to the direct-deps block. Lint and tests still pass, so it is cosmetic.

### README emoji is pre-existing

`README.md` still contains `✨sparkles✨` — but that line is inherited from the template and not introduced by this diff, so not a review blocker. Markus may want to strip it regardless.
